# 京津冀 PM2.5 预测与气象贡献研究界面

该目录为交互式研究界面的轻量运行目录，用于呈现京津冀 PM2.5 浓度预测、分时期模型结果和气象贡献分析。界面内嵌训练后的 LightGBM 模型，可选择城市、日期、小时和模型口径，输入气象条件后给出 PM2.5 预测结果，并呈现模型指标、SHAP 解释贡献、疫情分时期对比和气象背景图。

## 研究数据

- 研究区域：京津冀 13 个城市。
- 数据时间：2018-01-01 00:00 至 2026-05-31 18:00。
- 样本规模：958,711 条 city-hour 记录。
- PM2.5 缺失率：0%。
- ERA5 PBLH 缺失率：0%。
- 时期划分：疫情前 2018-2019、疫情期 2020-2022、疫情后 2023+。

2018-2022 PM2.5 使用 CNEMC/quotsoft 城市小时空气质量数据；2023 年以后使用此前整合的数据源。跨时期解释时应说明该数据源差异。

## 界面内容

- 预测台：选择城市、日期、小时和模型，输入温度、湿度、风速、风向、气压、PBLH、逆温和 PM2.5 时滞等变量，确认后输出当前小时 PM2.5 预测结果，并呈现城市历史 PM2.5/PBLH、季节参考、风场和当天气象变量剖面。
- 模型介绍：说明全时期高精度模型、分时期高精度模型、基础气象归因模型和过程型气象贡献模型的指标、用途、特征口径、SHAP 贡献和残差结果。
- 训练策略：按高精度预测模型和气象贡献模型分别说明变量准入、时间切分、特征生成、LightGBM/Optuna 调参、目标变量选择和评估口径。
- 研究验证：呈现留城市验证、透明样条对照、天气型机制、SHAP 稳定性和典型气象条件误差。

## 代表模型指标

| 模型 | 训练口径 | 测试 MAE | 测试 RMSE | 测试 R2 |
|---|---|---:|---:|---:|
| 全时期高精度预测模型 | 2018+ extended | 7.72 | 12.06 | 0.917 |
| 疫情前高精度预测模型 | 2018-2019 extended | 2.27 | 3.60 | 0.993 |
| 疫情期高精度预测模型 | 2020-2022 extended | 1.77 | 3.09 | 0.993 |
| 疫情后高精度预测模型 | 2023+ extended | 1.33 | 2.59 | 0.997 |
| 疫情前过程型气象贡献模型 | 2018-2019 weather-only, log1p(PM2.5) | 19.47 | 27.98 | 0.552 |
| 疫情期过程型气象贡献模型 | 2020-2022 weather-only, PM2.5 原值 | 13.31 | 21.40 | 0.674 |
| 疫情后过程型气象贡献模型 | 2023+ weather-only, 气候态异常 | 17.61 | 24.92 | 0.734 |

高精度模型用于预测能力评估；过程型气象贡献模型严格保持 weather-only 变量准入原则，更适合支撑大气扩散条件与污染浓度变化关系的研究讨论。

## 本地运行

```bash
pip install -r requirements.txt
streamlit run streamlit_app.py --server.port 8520 --server.address 127.0.0.1
```

## 目录说明

```text
app/                 研究界面源码
app_assets/          指标、SHAP、城市画像和界面数据
models/              已训练模型对象
streamlit_app.py     运行入口
requirements.txt     运行依赖
```

完整原始数据和下载脚本不放入该轻量运行目录；研究界面运行只依赖模型对象和轻量数据资产。
