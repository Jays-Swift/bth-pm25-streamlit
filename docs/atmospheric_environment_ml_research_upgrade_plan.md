# 京津冀 PM2.5 项目向大气环境机器学习研究升级方案

## 1. 研究定位

本项目不应仅定位为“基于机器学习的 PM2.5 浓度预测”，而应进一步提升为：

> 基于机器学习气象归一化与可解释建模的京津冀 PM2.5 气象贡献及疫情期差异研究。

项目的核心学术目标是利用机器学习模型刻画气象条件对 PM2.5 浓度变化的解释能力，并比较疫情前、疫情期、疫情后不同阶段中气象因子贡献结构的差异。

建议将研究主线概括为：

- 在严格变量控制条件下，定量评估气象条件对京津冀 PM2.5 变化的独立解释力；
- 比较疫情前、疫情期、疫情后三个阶段的气象贡献差异；
- 识别低边界层、弱风、高湿、南风输送、逆温和通风条件等关键气象过程对污染累积的影响；
- 通过 SHAP、条件误差分析、天气型分组和稳定性检验，提高模型解释结果的大气环境学可信度。

## 2. 核心研究问题

建议将研究问题从单纯预测扩展为以下几个方面：

1. 疫情前、疫情期、疫情后，气象条件对 PM2.5 的解释力是否存在阶段性差异；
2. PBLH、湿度、气压、风输送、逆温、通风系数等气象因子的贡献排序在不同时期是否稳定；
3. 静稳高湿、低边界层、南风输送、北风清洁输送、降水清除等天气型下，PM2.5 浓度和模型误差是否存在系统差异；
4. 高精度预测模型与气象贡献模型之间的性能差距，能否反映污染持续性、共污染物和排放背景对短时预测的重要性；
5. 气象-only 模型在剔除污染时滞和共污染物后，仍能解释多少 PM2.5 浓度变化。

## 3. 模型体系设计

项目应保留两个模型体系，但必须在论文中明确二者的研究功能差异。

### 3.1 高精度预测模型

高精度预测模型用于展示项目的应用预测能力和机器学习框架的性能上限。

该模型可以使用：

- 当前小时气象变量；
- ERA5 PBLH；
- 逆温指数；
- U/V 风分量；
- PM2.5 前 1/3/24 小时时滞；
- PM2.5 滚动均值；
- PM10、NO2、CO、SO2、O3 等共污染物；
- AOD、dust 等辅助污染背景变量；
- 城市、月份、小时、季节周期等时空控制变量。

该模型适合用于：

- 预测台交互展示；
- 模型性能上限说明；
- 与气象-only 模型进行对照；
- 说明污染持续性和共污染物对短时预测的重要性。

该模型不适合作为气象因子独立贡献的主要证据，因为污染时滞和共污染物会吸收一部分排放、人为活动和污染过程信息。

### 3.2 气象贡献模型

气象贡献模型是论文中更具有学术价值的核心模型。

该模型应主动剔除：

- PM2.5 当前值；
- PM2.5 时滞；
- PM2.5 滚动均值；
- PM10、NO2、CO、SO2、O3 等共污染物；
- AOD、dust 等污染背景变量；
- 任何由目标变量直接派生的特征。

该模型应保留：

- 气温、露点、相对湿度、气压、降水、云量；
- 风速、风向、U/V 风分量；
- PBLH、低 PBLH 标记；
- 逆温指数；
- 南风输送、北风清洁输送；
- 通风系数；
- 连续弱风、连续高湿、连续低 PBLH 小时数；
- 气象变量时滞、滚动均值和累计量；
- 城市空间位置、月份、小时和季节周期变量。

该模型用于讨论：

- 气象条件本身对 PM2.5 的解释力；
- 不同时期气象贡献结构的差异；
- PBLH、湿度、风输送、气压滞后等气象过程的重要性；
- 疫情期人为活动变化背景下气象解释力的稳定性。

## 4. 训练策略升级

### 4.1 时间阻塞验证

PM2.5 具有明显时间连续性，随机切分会造成相邻小时泄漏。因此应采用时间阻塞验证：

- 训练集使用较早时间段；
- 验证集使用训练集之后的连续时间段；
- 测试集使用最后的连续时间段；
- 测试集不参与参数选择。

该方法能够更真实地检验模型对未来时段的泛化能力。

### 4.2 分时期训练

建议将数据划分为：

- 疫情前：2018-2019；
- 疫情期：2020-2022；
- 疫情后：2023+。

每个时期单独训练气象贡献模型，避免全时期模型把疫情期和非疫情期差异平均掉。

每个时期应分别报告：

- R2；
- RMSE；
- MAE；
- Bias；
- Top SHAP；
- 条件误差；
- 天气型误差；
- SHAP 稳定性。

### 4.3 留城市验证

为了证明模型学习到的是区域气象规律，而不是简单记住城市差异，建议增加留城市验证。

基本做法：

- 每次留出一个城市作为测试城市；
- 其余城市用于训练和验证；
- 重复 13 次；
- 统计各城市外推表现。

该验证可以回答：

> 模型是否具备跨城市泛化能力。

如果留城市验证表现合理，项目的学术说服力会明显增强。

### 4.4 天气型分层验证

气象贡献模型不应只报告总体 R2 和 RMSE，还应在典型气象条件下单独评估。

建议分层条件包括：

- 低 PBLH；
- 弱风；
- 高湿；
- 低通风系数；
- 南风输送；
- 北风清洁输送；
- 降水清除；
- 逆温条件；
- 静稳高湿天气型。

每一类条件下单独报告模型误差，有助于判断模型在关键污染气象过程中的解释稳定性。

## 5. 天气型聚类分析

天气型分析是将项目提升为大气环境机器学习研究的重要环节。

建议使用气象变量聚类，构建若干典型天气型：

1. 静稳高湿型；
2. 低边界层弱风型；
3. 南风输送型；
4. 北风清洁输送型；
5. 高边界层扩散型；
6. 降水清除型。

每类天气型应分析：

- 出现频率；
- 疫情前、疫情期、疫情后的频率变化；
- PM2.5 中位数和分布；
- 模型误差；
- Top SHAP 特征；
- 典型气象机制。

该部分可以把机器学习结果转化为大气环境学解释，而不只是模型分数展示。

## 6. 气象过程指标体系

为了增强模型解释的物理意义，应系统构建气象过程指标。

### 6.1 边界层与垂直扩散

- PBLH；
- 低 PBLH 标记；
- 过去 24 小时平均 PBLH；
- 连续低 PBLH 小时数；
- 逆温指数；
- 逆温叠加强度。

解释意义：

低边界层和逆温会限制垂直混合空间，使污染物更容易在近地层累积。

### 6.2 水平输送

- 风速；
- U 风分量；
- V 风分量；
- 南风输送强度；
- 北风清洁输送强度；
- 风速 1/3/6/24 小时时滞；
- 过去 24 小时平均风速。

解释意义：

京津冀区域中，南风可能带来区域污染输送，北风通常有利于清洁空气输入和污染扩散。

### 6.3 湿度与二次生成环境

- 相对湿度；
- 露点；
- 连续高湿小时数；
- 湿度-PBLH 交互；
- 高湿弱风复合指标。

解释意义：

高湿环境有利于颗粒物吸湿增长，也可能促进二次无机气溶胶生成。

### 6.4 通风与静稳

- 通风系数：`PBLH * wind_speed`；
- 逆通风系数；
- 静稳指数；
- 连续弱风小时数；
- 低 PBLH + 弱风 + 高湿复合条件。

解释意义：

低通风条件通常意味着污染物水平输送和垂直扩散能力均较弱。

### 6.5 降水清除

- 降水量；
- 降水持续时间；
- 过去 6/12/24 小时累计降水；
- 湿清除指数。

解释意义：

降水能够通过湿清除过程降低颗粒物浓度，但其影响可能存在时间滞后。

## 7. SHAP 解释升级

单次 SHAP 排名不足以支撑稳健学术结论，应增加稳定性分析。

建议输出：

- 分时期 SHAP 排名；
- 分城市 SHAP 排名；
- 分天气型 SHAP 排名；
- bootstrap SHAP 均值；
- bootstrap SHAP 标准差；
- 95% 置信区间；
- Top 特征排名稳定性；
- 疫情前、疫情期、疫情后贡献差异比较。

解释时应避免将 SHAP 直接表述为因果贡献。更合适的表述是：

> SHAP 结果反映模型在给定特征空间下对 PM2.5 变化的解释贡献，不等同于严格因果效应。

## 8. 机制解释框架

机器学习结果必须落回大气环境机制。

建议按以下逻辑解释主要变量：

- PBLH 低：垂直扩散空间受限；
- 逆温增强：热力稳定层结抑制垂直混合；
- 高湿：颗粒物吸湿增长和二次生成条件增强；
- 弱风：水平扩散能力减弱；
- 南风输送：京津冀南部及上风向污染输入增强；
- 北风输送：清洁冷空气输入和污染扩散增强；
- 通风系数低：垂直和水平扩散能力共同减弱；
- 降水：湿清除降低颗粒物浓度；
- 气压滞后：天气系统转换和静稳过程的综合信号。

论文写作中应将模型解释结果和上述物理机制对应，而不是只陈述“某变量 SHAP 值较高”。

## 9. 不确定性与局限性

为了体现研究严谨性，应主动说明以下局限：

- SHAP 是模型解释结果，不等同于严格因果效应；
- 气象贡献模型剔除了污染时滞和共污染物，因此精度低于高精度模型是合理现象；
- 疫情期人为活动变化没有直接排放清单支撑，因此不能直接断言排放变化的因果贡献；
- 2023+ PM2.5 数据源与 2018-2022 不完全一致时，跨时期比较需要谨慎；
- 模型结果依赖特征构造和时间切分方式，应通过时间阻塞验证、留城市验证和 bootstrap 稳定性检验增强可信度。

建议最终论文或报告至少包含以下图表：

## 11. 推荐执行优先级

### 第一优先级

- 完善气象-only 归因模型；
- 固定疫情前、疫情期、疫情后分时期训练；
- 输出分时期 SHAP；
- 输出 bootstrap SHAP 置信区间；
- 完成天气型聚类。

### 第二优先级

- 增加留城市验证；
- 增加时间阻塞交叉验证；
- 增加天气型条件误差；
- 增加低 PBLH、弱风、高湿、南风输送条件下的模型误差分析。

### 第三优先级

- 加入 PDP 或 ALE；
- 增加典型污染过程案例；
- 若有条件，引入排放清单或活动强度数据作为对照；
- 将模型解释结果整理为论文式图表。

## 12. 最终研究表述建议

本项目最终可以总结为：

> 本研究基于 2018 年以来京津冀城市小时尺度 PM2.5 与 ERA5 气象资料，构建高精度预测模型和气象-only 贡献模型。通过分时期训练、气象过程特征构建、天气型分层、时间阻塞验证和 SHAP bootstrap 解释，定量分析边界层高度、湿度、气压、通风条件、逆温和区域输送对 PM2.5 变化的阶段性影响。研究重点不只在于提高预测精度，而在于揭示疫情期与非疫情期气象贡献结构的差异及其大气环境机制。
